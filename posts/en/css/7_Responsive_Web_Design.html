7. Responsive Web Design (RWD)

In the past, companies used to build entirely separate websites for mobile devices (like m.example.com). Today, Responsive Web Design (RWD) is the undisputed standard. RWD allows a single HTML source to fluidly adapt its layout to perfectly fit any screen size. The core technologies driving this are Media Queries and relative units.

Branching with Media Queries (@media)

Media queries detect conditions like the browser window's width or height and apply specific CSS rules only when those conditions are met. In the industry, the "Mobile First" approach is standard: you write your base styles for mobile screens, and progressively add complexity as the screen gets wider.

/* Base Styles (Mobile First) */
.container {
  display: flex;
  flex-direction: column;
}

/* Tablet (Screens 768px and wider) */
@media (min-width: 768px) {
  .container {
    flex-direction: row;
  }
}

/* Desktop (Screens 1024px and wider) */
@media (min-width: 1024px) {
  .container {
    max-width: 1000px;
    margin: 0 auto; /* Center the layout */
  }
}

Moving Beyond Pixels (px): Strategic Use of Relative Units

For a truly responsive design, relying purely on fixed pixels (px) is an anti-pattern. You must strategically use relative units that adapt to their surroundings.

  1. rem and em (The Core of Typography)
    • em: Relative to the font-size of its parent element. (If parent is 16px, 2em is 32px). It can cause compounded calculation headaches when deeply nested.
    • rem: Relative to the font-size of the root element (HTML). Since the browser default is usually 16px, it provides highly intuitive scaling. It is the most widely used unit for font sizes, padding, and margins in modern CSS.
  2. vw and vh (Filling the Viewport)
    • vw (Viewport Width): 1% of the browser's width. 100vw spans the exact full width of the screen.
    • vh (Viewport Height): 1% of the browser's height. 100vh is incredibly useful for creating full-screen landing sections.
