1. The Box Model and box-sizing

When a web browser renders elements on a screen, it treats every single HTML element as an invisible rectangular box. This core concept is known as the CSS Box Model.

The Four Areas of the Box Model

Every element is composed of the following four areas:

  1. Content: The innermost part where your text, images, or other actual content resides.
  2. Padding: The transparent space between the content and the border. The element's background color extends into this area.
  3. Border: The line that goes around the padding and content. You can style its thickness, color, and pattern.
  4. Margin: The outermost transparent space outside the border, used to push other elements away.

The Pitfall of the Default Box Model (content-box)

By default, the CSS box-sizing property is set to content-box. In this state, the width and height you assign to an element apply only to the Content area.

.box {
  width: 200px;
  padding: 20px;
  border: 5px solid black;
}

If you look at the code above, is the actual width of the box 200px? No. The actual rendered width becomes 200px (Content) + 40px (Left/Right Padding) + 10px (Left/Right Border) = 250px. This makes calculating layouts incredibly frustrating and often leads to elements breaking out of their parent containers.

The Savior: box-sizing: border-box;

To solve this headache, modern web development almost universally includes the following reset at the beginning of a project:

*, *::before, *::after {
  box-sizing: border-box;
}

When you apply border-box, the width and height you set will encompass the Content + Padding + Border combined. So, if you declare width: 200px;, the total width remains exactly 200px regardless of how much padding or border you add. The content area simply shrinks to accommodate them. It completely changes the game by making CSS sizing intuitive and predictable.
