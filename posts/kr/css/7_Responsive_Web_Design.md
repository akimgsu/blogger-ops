# 7. 반응형 웹 디자인 (Responsive Web Design)

과거에는 모바일용 사이트(`m.example.com`)와 데스크톱용 사이트를 따로 만들었지만, 이제는 하나의 HTML 소스로 모든 기기의 화면 크기에 맞게 레이아웃이 물 흐르듯 변하는 **반응형 웹 디자인(RWD)**이 표준입니다. 이를 구현하기 위한 핵심 기술이 바로 미디어 쿼리와 가변 단위입니다.

## 미디어 쿼리 (@media) 분기 처리

미디어 쿼리는 브라우저 화면의 너비나 높이 같은 조건을 감지하여 특정 조건에서만 CSS가 작동하도록 돕습니다. 실무에서는 보통 모바일 화면을 먼저 디자인하고(Mobile First), 화면이 커질수록 살을 붙여나가는 방식을 선호합니다.

```css
/* 기본 스타일 (모바일 우선 적용) */
.container {
  display: flex;
  flex-direction: column;
}

/* 태블릿 (가로가 768px 이상일 때) */
@media (min-width: 768px) {
  .container {
    flex-direction: row;
  }
}

/* 데스크톱 (가로가 1024px 이상일 때) */
@media (min-width: 1024px) {
  .container {
    max-width: 1000px;
    margin: 0 auto; /* 중앙 정렬 */
  }
}
```

## 픽셀(px)을 넘어: 가변 단위의 전략적 사용

반응형 디자인에서는 화면 크기에 따라 유연하게 변하는 상대 단위를 사용하는 것이 중요합니다.

1. **`rem`과 `em` (타이포그래피의 핵심)**
   - `em`: **자신의 부모 요소**의 폰트 크기를 기준으로 배수 계산. (부모가 16px이면 2em은 32px) 중첩될수록 계산이 복잡해지는 단점이 있습니다.
   - `rem`: **최상위 루트 요소(HTML)**의 폰트 크기를 기준으로 배수 계산. (보통 브라우저 기본 16px). 계산이 매우 직관적이므로 실무에서 폰트, 마진, 패딩에 가장 널리 쓰이는 단위입니다.
2. **`vw`와 `vh` (화면 꽉 채우기)**
   - `vw` (Viewport Width): 브라우저 너비의 1%. `100vw`는 화면 전체 너비.
   - `vh` (Viewport Height): 브라우저 높이의 1%. `100vh`는 화면 전체 높이. 풀스크린 섹션을 만들 때 유용합니다.

---
### Chapter Summary
- 반응형 웹 디자인은 하나의 HTML로 다양한 화면 크기에 대응하는 현대 웹의 표준입니다.
- `@media` 쿼리를 사용해 화면 너비(Breakpoint)를 기준으로 레이아웃과 스타일을 다르게 적용합니다.
- 고정된 `px` 대신 `rem`, `vw`, `vh` 같은 가변 단위를 사용하면 화면 크기에 맞게 자연스럽게 조절되는 UI를 만들 수 있습니다.
