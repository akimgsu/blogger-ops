# 2. CSS 명시도와 계단식 덮어쓰기 (Specificity & Cascade)

CSS(Cascading Style Sheets)라는 이름에서 알 수 있듯이, 스타일은 **계단식(Cascade)**으로 위에서 아래로 흐르며 적용됩니다. 하지만 여러 스타일이 동일한 요소를 가리킬 때, 브라우저는 **명시도(Specificity)**라는 점수 시스템을 통해 어떤 스타일을 최종적으로 적용할지 결정합니다.

## 명시도(Specificity) 점수 계산법

CSS 선택자의 명시도는 4자리의 점수로 계산할 수 있습니다: `(인라인, ID, Class, 태그)`

1. **인라인 스타일 (1, 0, 0, 0)**: HTML 요소 내부에 직접 작성한 `style=""` 속성. 가장 강력합니다.
2. **ID 선택자 (0, 1, 0, 0)**: `#header` 등 ID를 통한 선택자.
3. **Class, 가상 클래스, 속성 선택자 (0, 0, 1, 0)**: `.btn`, `:hover`, `[type="text"]` 등.
4. **태그, 가상 요소 선택자 (0, 0, 0, 1)**: `div`, `p`, `::before` 등.

예를 들어 `.card .title`의 점수는 Class가 2개이므로 `(0, 0, 2, 0)`입니다. 반면 `#title`은 ID가 1개이므로 `(0, 1, 0, 0)`입니다. 점수가 더 높은 쪽이 무조건 이기게 됩니다. 점수가 동일하다면? **나중에 선언된(아래쪽에 위치한) 스타일**이 덮어씁니다.

## !important의 유혹과 위험성

`!important`는 모든 명시도 점수를 무시하고 최우선으로 스타일을 강제 적용하는 치트키입니다.

```css
.btn {
  background-color: blue !important;
}
```

초보자들은 스타일이 덮어써지지 않을 때 `!important`를 남발하는 유혹에 빠지곤 합니다. 하지만 이를 남용하면 CSS의 자연스러운 Cascade 흐름이 깨지고, 나중에 스타일을 수정해야 할 때 더 많은 `!important`를 추가해야 하는 유지보수의 악몽(Hell)이 시작됩니다.

**충돌 해결 방식의 정석**:
스타일이 적용되지 않는다면 `!important`를 쓰기 전에 개발자 도구를 열어 **어떤 선택자가 더 높은 점수(Weight)를 가지고 있는지 확인**하세요. 부모 클래스를 하나 더 추가하여 점수를 자연스럽게 높이거나, 중복되는 구조를 리팩토링하는 것이 올바른 해결책입니다.

---
### Chapter Summary
- CSS는 명시도(Specificity) 점수와 선언 순서(Cascade)를 기준으로 적용 우선순위를 결정합니다.
- 점수 가중치는 **인라인 스타일 > ID > Class > 태그** 순으로 강력합니다.
- `!important`는 유지보수를 어렵게 만드는 안티 패턴이므로 남용을 피하고, 선택자의 점수를 조정하여 충돌을 해결해야 합니다.
